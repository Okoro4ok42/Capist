# capist

Кроссплатформенная консольная утилита на C++17, которая превращает произвольное изображение в текстуру плаща для клиента [ruBeta](https://mc.rubeta.net) (кастомный клиент Minecraft Beta 1.7.3).

## Зачем это нужно

ruBeta принимает текстуры плащей только в виде PNG строго определённого размера. capist берёт любое изображение, приводит его к нужному формату и собирает итоговый файл автоматически — руками в редакторе делать это неудобно и легко ошибиться в геометрии.

## Формат текстуры ruBeta

Итоговый PNG всегда имеет размер **352×272** — это фиксированный размер, который требует клиент. Внутри этого холста реально используется только левый верхний угол:

```
┌──────────────┬──────────────┬─────────────────────────┐
│              │              │                          │
│   64×75      │   64×75      │                          │
│  оригинал    │   зеркало    │      прозрачная зона      │
│  (уменьшен)  │  по горизон- │   (не используется, но    │
│              │      тали    │    обязана присутствовать) │
├──────────────┴──────────────┘                          │
│                    352×272 холст                        │
└──────────────────────────────────────────────────────────┘
```

- Исходное изображение сжимается до **64×75** и кладётся в левый верхний угол.
- Рядом с ним, вплотную справа, кладётся его горизонтально отражённая копия того же размера.
- Всё остальное пространство холста (до 352×272) остаётся полностью прозрачным — это обязательное поле формата, не визуальный контент.

Все эти числа — не магические константы «из ниоткуда»: `kTileWidth`/`kTileHeight` (64×75) и `kCanvasWidth`/`kCanvasHeight` (352×272) вынесены в именованные константы в начале `main.cpp` с комментарием, объясняющим происхождение каждого числа.

## Сборка

Утилита не использует ничего платформенного — только stb_image / stb_image_write (лежат в `src/`) и стандартную библиотеку C++17. Собирается одинаково на Windows, Linux и macOS.

```bash
g++ src/main.cpp -std=c++17 -O2 -o capist
```

На Windows это тоже работает как есть — через MinGW/MSYS2, либо через MSVC:

```bash
cl /EHsc /std:c++17 src\main.cpp
```

Готовые сборки для Windows/Linux можно взять на вкладке [Releases](../../releases), не собирая проект самостоятельно.

## Использование

```bash
./capist <входное-изображение> [выходной.png]
```

- `<входное-изображение>` — любой формат, который понимает stb_image (PNG, JPG, BMP, GIF, TGA и другие).
- `[выходной.png]` — необязательный путь для результата. Если не указан, файл сохраняется рядом с исходным как `<имя>_c(r)apist.png`.

Пример:

```bash
./capist my_art.png cloak.png
```

## Как это устроено

1. `stbi_load` читает исходное изображение и приводит его к RGBA.
2. Билинейное уменьшение до 64×75 (собственная реализация, без внешних зависимостей вроде stb_image_resize).
3. Пустой прозрачный холст 352×272, в который вставляется уменьшенное изображение и его зеркальная копия.
4. `stbi_write_png` сохраняет результат.
